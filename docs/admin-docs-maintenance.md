# Поддержка админ-документации (`/admin/docs`)

Пользовательская документация для организаторов живёт в **`frontend/app/pages/admin/docs.vue`**. Этот файл — единственный источник текста на странице `/admin/docs`.

## Когда обновлять

Обновляйте `docs.vue`, если меняется:

- поведение справочников (валидации, привязки к турниру/команде);
- шаги мастера создания или мастера запуска турнира;
- вкладки карточки турнира (инфраструктура, заявки, протокол);
- уведомления (Telegram / Email, новые события);
- матрица ролей или тарифов;
- типичные сценарии (детская лига, кубок, фестиваль).

Внутренние планы и roadmap — в `docs/*.md` (например `tournament-creation-scenarios-plan.md`). В user guide переносите только **актуальное для организатора**, без технических кодов и статусов разработки.

## Структура `docs.vue`

| Блок | Назначение |
|------|------------|
| `toc` | Оглавление (десктоп + мобильный drawer) |
| `<article id="...">` | Разделы; `id` должен совпадать с пунктами `toc` |
| `docs-h2` / `docs-h3` | Заголовки; у `h3` — свой `id` для якорей |

При добавлении раздела: пункт в `toc` → `<article>` или `<h3 id="...">` → проверка скролла по якорю.

## Чеклист перед merge (фича затрагивает организатора)

1. Есть ли новое поле/вкладка в турнире? — описать в «Турниры: как сделать правильно» или «Справочники».
2. Меняется ли валидация (возраст, категория, заявки)? — таблица «что проверяется» в справочниках.
3. Новое уведомление? — раздел «Уведомления».
4. Убрать устаревшее из «Идеи улучшений», если фича уже в проде.
5. Дата в комментарии внизу `docs.vue` (опционально) или в commit message.
