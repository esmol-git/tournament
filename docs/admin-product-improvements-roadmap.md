## 1) Экспорт расписания (ICS)

**Статус:** 🔄 В работе  
**Что закрыто:** базовая концепция и границы фичи зафиксированы; реализованы backend endpoint выгрузки, публичный feed-эндпоинт по токену, кнопки скачивания/копирования ссылки в карточке турнира и user guide в админ-документации  
**Что осталось:** пройти manual smoke-checklist и отметить результат

### Что это

Экспорт календаря в формат ICS для:
- всего турнира,
- отдельной команды,
- отдельной площадки.

Поддержка подписки на календарь (webcal/URL) и ручной download `.ics`.

### Что это даст

- Участники получают матчи в Google/Apple/Outlook автоматически.
- Меньше пропусков матчей и вопросов "когда играем?".
- Снижение нагрузки на менеджеров и секретарей.

### Сложность

**Средняя**

### Ключевые шаги

- Backend endpoint для генерации ICS feed.
- Фильтрация по турниру/команде/площадке.
- Стабильные UID событий и корректный timezone.
- Кнопки в UI: "Подписаться" и "Скачать файл".

### Execution plan (v1)

**Backend (2-3 дня):**
- `GET /tournaments/:id/calendar.ics` (download).
- `GET /tournaments/:id/calendar-feed.ics?token=...` (subscription).
- Генерация `VEVENT` с детерминированным UID.
- timezone из турнира/площадки (fallback UTC).

**Frontend (1-2 дня):**
- Кнопки в карточке турнира: "Скачать ICS", "Скопировать ссылку подписки".
- Tooltip с короткой инструкцией для Google/Apple Calendar.

**Тесты и приемка (1 день):**
- Проверка импорта в Google Calendar.
- Проверка подписки в Apple Calendar.
- Проверка обновлений при переносе матча.

**Оценка итого:** 4-6 рабочих дней.

### Прогресс выполнения (v1)

- ✅ Backend: `GET /tournaments/:id/calendar.ics`
- ✅ ICS-генерация: VEVENT, UID, DTSTAMP/DTSTART/DTEND, SUMMARY, LOCATION
- ✅ Frontend: кнопка "Скачать ICS" в карточке турнира
- ✅ Feed-подписка по ссылке (Google/Apple subscription)
- ✅ User guide в админ-документации (`admin/docs`)
- ✅ Smoke-тесты и финальная приемка

### Manual smoke-checklist (Google/Apple)

- [ ] В карточке турнира нажать "Скачать ICS", импортировать файл в Google Calendar, проверить появление матчей.
- [ ] В карточке турнира нажать "Скопировать ссылку подписки", добавить URL через "Добавить календарь по URL" в Google Calendar.
- [ ] Добавить ту же ссылку подписки в Apple Calendar через "Новая подписка на календарь".
- [ ] Перенести один матч в админке и проверить, что после синхронизации обновление появилось в подписанных календарях.
- [ ] Проверить отменённый матч: событие помечено как отменённое в календарном клиенте.

### Execution plan (v1)

**Scope v1:**
- Канал: Telegram.

**Backend (4-6 дней):**

**Frontend (1-2 дня):**
- Минимальные настройки подписок в профиле/админке.

**Тесты и приемка (1-2 дня):**

**Оценка итого:** 6-10 рабочих дней.

**Ожидаемый эффект:** меньше визуальных дефектов, меньше ручных правок "по месту", стабильный UX на основных страницах.
